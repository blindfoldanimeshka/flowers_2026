## Задача: внутренняя прокрутка фото в карточке товара (до 3 фото)

### Цель
Сделать в карточке товара “внутренний просмотр галереи”:
- **ПК**: при наведении курсором на область фото (слева направо) переключаются изображения.
- **Мобилка**: переключение делается **свайпом** (влево/вправо) по области фото.
- Отображать **не больше 3 фото** на один товар (если у товара больше — берём первые 3).
- Показывать **индикатор текущего фото** (например, точки/полоски) внизу области фото (как на референсе).

---

## Что использовать (с точки зрения компонентов)
В карточке товара должна быть:
1. **Состояние активного фото** (например, `activeImageIndex`), где значение в диапазоне `0..(imagesCount-1)`.
2. **Источник изображений**:
   - взять исходный массив `imageGallery` (если он есть) + `imageSrc` (как fallback),
   - убрать дубликаты (если встречаются),
   - применить лимит: `images.slice(0, 3)`,
   - если после этого изображений 0 — использовать fallback (например, `imageSrc`/дефолт).
3. **Область** (контейнер) фиксированной геометрии под фото:
   - контейнер должен сохранять пропорции (через `aspect-ratio` или фиксированную высоту),
   - картинка рендерится “как фон” (`object-fit: cover`), чтобы не вытягивалась.

---

## Десктоп: переключение при hover слева направо
### Условия активации
- Активировать только когда у устройства есть hover (например, через `matchMedia('(hover: hover) and (pointer: fine)')`).
- Если изображений `1` — не навешивать интерактивность, индикатор не показывать или показывать только 1 вариант.

### Поведение
- Слушать событие **`onMouseMove`** (или аналог) на контейнере фото.
- Для каждого `mousemove`:
  - взять `boundingClientRect()` контейнера,
  - вычислить `x` — позицию курсора относительно левого края,
  - вычислить отношение `ratio = clamp(x / width, 0..1)`,
  - определить `nextIndex = floor(ratio * images.length)`,
  - гарантировать, что `nextIndex` не выходит за `images.length - 1` (краевой случай при `x == width`).
- При выходе курсора из области:
  - вернуть `activeImageIndex` в `0` (или оставить текущий — но лучше как на референсе: сброс).

### Важные нюансы
- Не использовать переключение “на каждой пиксельной разнице” через лишние вычисления/рендеры, но обычно state update при каждом `mousemove` допустим при небольших массивах (до 3).
- Переключение должно выглядеть плавно и без “дергания”: контейнер должен быть того же размера, что и при текущем изображении (фиксированная геометрия).

---

## Мобилка: переключение свайпом
### Условия активации
- Активировать для touch-устройств (можно просто не включать hover-логику при `canUseHover === false`).
- Если изображений `1` — свайп не нужен.

### События
- Использовать `onTouchStart` и `onTouchEnd` (минимально).
  - `touchstart`: запомнить `startX` (и опционально `startY`).
  - `touchend`: взять `endX`, посчитать `deltaX = endX - startX`.

### Логика
- Задать порог переключения:
  - `thresholdPx` (например, 30–40px) или
  - порог как доля ширины контейнера (например, 0.2 * width).
- Если `abs(deltaX) < threshold` — не переключать фото.
- Если `deltaX < 0` (свайп влево) — перейти к следующему фото:
  - `activeImageIndex = min(activeIndex + 1, lastIndex)`
- Если `deltaX > 0` (свайп вправо) — перейти к предыдущему:
  - `activeImageIndex = max(activeIndex - 1, 0)`

### Что делать с вертикальным скроллом страницы
- Если пользователь свайпает вертикально, переключение не должно мешать.
- Чтобы не “перехватывать” жесты:
  - опционально сравнить `abs(deltaY)` и `abs(deltaX)` и если вертикальная составляющая больше — игнорировать переключение.
- Снаружи модалки/карточки не ломать общий скролл (интерактивность только в рамках контейнера фото).

### Индикатор
- Обновлять индикатор при изменении `activeImageIndex`.

---

## Индикатор текущего фото (как на референсе)
### Визуальное требование
- Разместить индикатор **снизу** внутри области фото.
- Сделать “активный” слайд заметнее (больше ширина/яркость).
- На `images.length <= 1` индикатор можно скрыть.

### Логика
- Для каждого слайда `index`:
  - если `index === activeImageIndex` — показывать активный стиль,
  - иначе — неактивный стиль.

---

## Ограничения: максимум 3 фото на товар
- Даже если входной `imageGallery` содержит больше изображений — отображать только 3.
- Индикатор и переключение должны работать строго по фактическому числу отображаемых фото (1..3).

---

## Edge cases (обязательные случаи)
1. `imageGallery` пустой:
   - галерея должна состоять минимум из `imageSrc` (fallback).
2. В `imageGallery` есть дубликаты:
   - важно убрать дубликаты, чтобы индикатор и переключение не “прыгали”.
3. Изображение не загрузилось:
   - должен быть fallback на дефолтное изображение.
4. Одно фото:
   - hover/swipe отключаются, индикатор скрывается.

---

## Критерии приемки (как проверить)
- ПК:
  - наводишь курсор слева направо по области фото → меняется изображение слева направо;
  - выходишь с курсора → возвращается на первое фото (если так задумано);
  - индикатор показывает текущий кадр.
- Мобилка:
  - свайпаешь влево → переход на следующее фото;
  - свайпаешь вправо → возврат на предыдущее;
  - вертикальный скролл страницы не ломается.
- Для товара с 4+ фото:
  - показываются только 3, переключение и индикатор работают по ним.
