## 📋 Финальная проверка

### Перед деплоем:
- [ ] Все тесты проходят

- [ ] Health check работает
- [ ] Бэкап создан
- [ ] Переменные окружения настроены

### После деплоя:
- [ ] Сайт доступен по HTTPS
- [ ] Все страницы загружаются

- [ ] Корзина и заказы функционируют
- [ ] Админ-панель доступна
- [ ] Логи не содержат ошибок

## 🚨 Критические моменты

### Обязательно проверить:
1. **Безопасность** - все секреты сгенерированы и настроены
2. **База данных** - подключение работает, бэкапы настроены
3. **SSL/HTTPS** - сайт доступен по HTTPS
4. **Мониторинг** - health check работает
5. **Логирование** - ошибки логируются

### Частые проблемы:
- Неправильные переменные окружения
- Отсутствие SSL сертификата
- Неправильные права доступа к файлам
- Проблемы с подключением к MongoDB
- Отсутствие бэкапов

## 📞 Экстренные контакты

При возникновении проблем:
1. Проверьте логи: `docker-compose logs -f app`
2. Проверьте health check: `curl https://your-domain.com/api/health`
3. Проверьте переменные окружения
4. Проверьте подключение к базе данных
5. Создайте бэкап перед любыми изменениями

## 🔄 Обновления

### Для обновления приложения:
```bash
# Остановка сервисов
docker-compose down

# Обновление кода
git pull origin main

# Пересборка и запуск
docker-compose up -d --build

# Проверка
npm run health-check
```
